Python is the language of data. ## Scalable Serverless Engine

Python alone isn't enough. Scaling up compute on demand normally means deploying a new Docker container. Fused lets you create User Defined Functions — simple, self-contained Python functions that run on our serverless compute infrastructure. All you need is a `@fused.udf` decorator:

```python
@fused.udf
def udf():
    import pandas as pd
    return pd.read_parquet("s3://fused-sample/demo_data/airbnb_listings_sf.parquet")
```

You can call thousands of jobs at the same time:

```python
@fused.udf
def udf():
    basic_example = fused.load('process_data')  # Load existing UDF
    results = basic_example.map(range(100_000), max_workers=500)  # Run in parallel
    return results.df()
```

This spins up as many nodes as needed under the hood.

<img src="/img/blog/2026-04-29-what-is-fused/fused-parallel.png" alt="Fused parallel job execution" style={{width: '75%', display: 'block', margin: '0 auto'}} />

To put that in perspective: two thousand 90-second jobs complete in 120 wall-clock seconds — just 30 seconds of total orchestration overhead, at 0.015s per job, each on a dedicated 4-core machine with 10 GB of RAM. A regular EC2 instance of 128 cores takes ~2 minutes just to turn on. Fused does the whole job in that time.

## Expose Everything as an API

Every project you build in Fused lives in a **Canvas** — a freeform board where every component is either a UDF running Python, a sticky note, or a widget to explore data.

<img src="/img/blog/2026-04-29-what-is-fused/canvas.png" alt="Fused Canvas" style={{width: '75%', display: 'block', margin: '0 auto'}} />

Each Canvas is shareable, and every component is directly accessible through an API call. Think of a Canvas as a directory of all your endpoints, and each UDF as an endpoint.

Every Canvas can also be turned into a **Fused MCP Server** — a list of all the endpoints in your Canvas, corresponding to all your UDFs:

<img src="/img/blog/2026-04-29-what-is-fused/mcp-server.png" alt="Fused MCP Server" style={{width: '75%', display: 'block', margin: '0 auto'}} />

Every UDF you create can instantly become a parameterized endpoint callable from anywhere (once you've scoped the right permissions):

## Scoped Security & Access Control

Every Fused project can be shared and turned into a Fused MCP server, giving you fine-grained control over who can access what:

- **Public** — Anyone can see and execute your UDFs. Optionally require a passcode.
- **Private** — Only members of your Fused team can access your work. You can issue temporary access tokens (e.g. valid for 1 hour) and we support service accounts for secure machine-to-machine delivery.

You control the scope of each UDF individually.

## Example: RAG Your Docs

We built a bot that answers questions about Fused documentation directly in Slack — all built inside Fused itself.

<img src="/img/blog/2026-04-29-what-is-fused/rag-slack.png" alt="RAG Slack bot built in Fused" style={{width: '75%', display: 'block', margin: '0 auto'}} />

It works in three steps:

- **Connect** — Crawl any docs site via its sitemap, then parallelise fetching and cleaning all pages into a single `llms.txt` file saved to S3.
- **Process** — Fetch the `llms.txt` file, split it into overlapping chunks, embed them with an off-the-shelf LLM, and write the vector embeddings to a LanceDB table.
- **Expose** — A `vector_search` UDF takes a natural language question, embeds it, and runs a cosine similarity lookup against the LanceDB table — returning the top 4 most relevant chunks, cached for 1 hour.

The result: any AI agent (or Slack bot) can query your docs with a single request:

```
https://udf.ai/fc_YOUR_SHARED_TOKEN/vector_search.json?question=what is fused?
```

All of this runs on S3, simple Python, smart caching, and parallel processing. No additional setup required — just log in and start building.

See the [full example in the docs](https://docs.fused.io/examples/rag-overture-docs).
