This example turns a spreadsheet sitting in **Google Drive** into a board-ready **infographic image** using an AI image model, then drops it into an existing **Google Slides** deck — all from one Fused Canvas. No screenshots, no manual chart-building, no leaving Fused.

![The full pipeline on one Canvas — Part 1, Part 2, Part 3](/img/examples/gdrive-to-slides-canvas.png)

*The whole pipeline on a single Canvas — Drive file picker on the left, infographic generation in the middle, Slides publishing on the right.*

## Try it out

[**Open the Fused Canvas →**](https://unstable.fused.io/canvas/fc_fused/gdrive_to_infographic)

:::tip Make a copy first
The Canvas references your own Google Drive and your own OpenRouter key, so you need to **fork it into your workspace** before running:
1. Open the Canvas above.
2. Click **⋮** → **Make a copy** to clone it into your account.
3. Open the copy — everything below assumes you're working in your own version.
:::

:::info Two things to set up before you run
- **Connect Google Drive** — Workbench → **Integrations & Secrets** → **Google Drive** → **Connect**, then use the Picker to grant access to the spreadsheet folder you want to read *and* the Slides decks you want to publish into. With the `drive.file` scope, Fused only sees files you explicitly grant — so **a deck won't appear in the picker on the last node until you've added it via the Picker**.
- **Add an OpenRouter API key** — get one from [OpenRouter](https://openrouter.ai/keys) (free credits available), then Workbench → **Preferences** → **Environment** → **Secrets management** ([docs](/workbench/preferences#secrets-management)) and add it as `openrouter_api_key`.
:::

## How the Canvas is organized

Three folders on the top row — one per stage — drive the demo. Two more folders at the bottom hold the supporting UDFs (file listing, image generation, Slides API) — you don't normally interact with those.

### Part 1 · Pick data from Google Drive

The first folder holds a single [widget](/guide/data-input-outputs/import-connection/widgets) — **`widget_file_picker`** — built from a couple of [dropdowns](/guide/data-input-outputs/import-connection/widgets#available-widgets) and an [sql-table](/guide/data-input-outputs/import-connection/widgets#available-widgets). Pick the folder you want, then pick a CSV or XLSX inside it, and a live preview of the first rows renders below. Fused's [Google Drive integration](/workbench/integrations/gdrive) streams the file straight into a `pandas` DataFrame in memory — nothing is copied to S3 or any intermediate store.

The actual file-listing and file-loading logic lives in the **list csv/excel files** folder at the bottom of the Canvas. You don't need to interact with those nodes — the widget calls them for you — but they're there if you want to poke at them, fork them, or wire them into another pipeline.

### Part 2 · Generate the infographic

The middle folder is where the AI work happens. The **`widget_style`** widget on the left gives you a [gallery picker](/guide/data-input-outputs/import-connection/widgets#available-widgets) of clean editorial style references (BBC News, FiveThirtyEight, dark mode, ggplot, minimal). The image you pick is sent to the model alongside the data so it mimics that style. There's also a text input for a custom style URL, and a title field wrapped in a [form](/guide/data-input-outputs/import-connection/widgets#available-widgets) — the form's submit button is what actually fires generation, so typing in the title doesn't auto-trigger anything until you click **Generate Infographic**.

On submit, **`Infographic_Generator`** on the right does the whole image step in one pass: it reads the data, asks an image model on OpenRouter (Gemini 2.5 Flash Image) for a chart that matches the style reference and the title, saves the PNG to your Google Drive, and shows you a live preview. Because the image lives in Drive, the next stage just needs the file's link — there's no second upload anywhere.

### Part 3 · Publish into a Google Slides deck

![Deck picker, insert index, Publish button, and a live preview of the new slide](/img/examples/gdrive-to-slides-step3.png)

The right folder holds **`widget_publish`**: a [dropdown](/guide/data-input-outputs/import-connection/widgets#available-widgets) of your Slides decks, a [number input](/guide/data-input-outputs/import-connection/widgets#available-widgets) for the insert position, and a Publish button. The deck dropdown and insert index are inside a form, so nothing fires until you submit — you can change your mind freely before committing.

When you click **Publish to Slides**, Fused inserts a new slide at the position you chose with the title from Part 2 and the infographic image as a full-bleed visual. After it lands, an embedded preview of the new slide renders right under the form so you can confirm what made it in.

:::tip "I don't see my deck in the dropdown"
Slides decks only appear in the picker once they've been explicitly granted via the Drive Picker (the `drive.file` OAuth scope is per-file, not per-drive). Open **Integrations & Secrets** → **Google Drive** → **Picker** and add the deck you want before refreshing the Part 3 widget.
:::

:::warning Let Part 2 finish before clicking Publish
Part 2 sets a canvas parameter (`generated_image_url`) once the infographic preview renders. If you click Publish *before* the preview shows up, that parameter is still empty and you'll see a slide get inserted with just the title and no image. Wait for the infographic to appear, then publish.
:::

The supporting Slides-API UDFs are grouped in the **find and insert slides** folder at the bottom of the Canvas. Same idea as the bottom-row folder for Part 1: you don't normally touch them, they just sit there for reference.

## What's actually getting called

A quick map of what powers each top-row folder, in case you want to fork and modify:

| Folder | What it does | Calls into |
|---|---|---|
| **Part 1 · Pick data from Google Drive** | Lists folders + files, streams the selected sheet into a DataFrame | `list csv/excel files` folder (Drive listing + CSV/XLSX loader) |
| **Part 2 · Generate infographic** | Builds a prompt from the data, calls an image model, saves the PNG to Drive | OpenRouter (image model) + Google Drive (upload + sharing) |
| **Part 3 · Publish to Google Slides** | Inserts a new slide containing the title and the Drive-hosted image | `find and insert slides` folder (Slides API + Drive search) |
