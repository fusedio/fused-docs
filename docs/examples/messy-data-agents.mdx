Real-world data rarely lives in one place. Supplier lists sit in Snowflake, store metadata in Google Sheets, and review scores land in S3 as CSVs. Manually consolidating these sources before every analysis is slow and brittle. With Fused, you can join disparate datasets inside a UDF pipeline and expose the combined result to AI agents — no manual consolidation required.

## How It Works

### 1. Connect multiple data sources

Each data source is loaded by its own UDF in the Canvas:

- **Snowflake** — supplier and inventory data
- **Google Sheets** — store metadata and locations
- **AWS S3** — review scores and ratings as CSV/Parquet files

### 2. Join datasets in Python

A downstream UDF joins the tables on a shared key (e.g. store ID), producing one clean, combined DataFrame. The heavy lifting — reading, filtering, and joining — stays inside Fused. The agent never sees raw intermediates.

### 3. Expose the joined data as an MCP tool

The Canvas publishes the joined result as an agent-callable endpoint. The tool description tells the agent what data is available and what parameters it accepts.

### 4. Ask business questions

Once connected, an AI agent (Claude Code, ChatGPT, or a custom chatbot) can query the combined dataset directly. For example:

- *"Which stores have the highest ratings?"*
- *"Show me the bottom-performing suppliers."*
- *"Compare average review scores across regions."*

The agent draws answers from all three sources at once — no manual data wrangling needed.

## Try It Out

Open the [Fused Canvas](https://unstable.fused.io/canvas/fc_3NzqyeXDOMTNZ5pm1Sm1v8?bounds=699%2C2148%2C1480%2C3744) to explore the live pipeline. The Canvas connects to all three data sources, joins them, and exposes the result to an AI agent — ready for you to query.
