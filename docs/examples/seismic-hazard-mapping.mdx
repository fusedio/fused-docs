_A complete example showcasing how to combine a seismic hazard raster with building vector data — using H3 hexagons as a universal join key — to find where the most buildings sit in the most dangerous earthquake zones._

:::tip Try it yourself
Open the live canvas: [Seismic Hazard Mapping](https://www.fused.io/canvas/fc_26RsEajoeO8MGcP6wJgkqm?bounds=-329%2C-1758%2C6459%2C3069)
:::

## The problem: where are buildings most at risk?

Italy sits at the intersection of the African and Eurasian tectonic plates, making it one of Europe's most seismically active countries. Historic cities like L'Aquila, Assisi, and Amatrice have suffered devastating earthquakes in living memory.

The goal of this canvas is to answer a practical question: **where is the highest concentration of buildings in high-seismic-hazard zones?**

To answer it we need two things:
- A **seismic hazard map** — how likely and how severe are ground shaking events at each location?
- A **building inventory** — where are buildings, and how much built area is there?

These come from two completely different data sources, in two completely different formats. This is where Fused makes the analysis elegant.

## Data sources

**Seismic hazard** comes from the [GEM Foundation's OpenQuake Global Seismic Hazard Map (GSHM 2023)](https://maps.openquake.org/map/gshm-2023-1/). OpenQuake is the reference platform used by governments and engineering firms worldwide for seismic risk assessment. The dataset is distributed as a GeoTIFF where each pixel stores a Peak Ground Acceleration (PGA) value — the expected ground motion for a 1-in-475-year event.

**Buildings** come from [Overture Maps](https://overturemaps.org/), one of the most comprehensive open building footprint datasets available. It covers all of Italy as pre-partitioned GeoParquet files, each holding the building polygons for a geographic partition.

:::note
The seismic hazard raster and the Overture building polygons have nothing in common structurally: different formats, different resolutions, different projections. A traditional workflow would require a spatial join — a slow, topology-error-prone operation. Here, we sidestep that entirely using H3.
:::

## Canvas overview

The canvas is built from a chain of composable UDFs. Each one does one job and passes its output directly to the next:

| Step | UDF | What it does |
|------|-----|--------------|
| 1 | `openquake_explorer` | Downloads the OpenQuake GeoTIFF and saves it to S3 once |
| 2 | `loading_seismic_data` | Reads the TIF for a given bounding box, returns a pixel array |
| 3 | `hexagonified_seismic_hazard` | Converts raster pixels → H3 hexagons, averaging hazard per cell |
| 4 | `Overture_partitions` | Shows Overture GeoParquet partitions across Italy to pick the right file |
| 5 | `Overture_to_hex_single_file` | Reads one Overture partition, aggregates building area per H3 hex via DuckDB |
| 6 | `join_over_buildings` | Inner-joins the two hex layers on the H3 cell ID |
| 7 | Dashboard widget | Scatter chart + KPI tiles + map to explore the joined result interactively |

Changing the H3 resolution (`res=6`) propagates automatically through the entire pipeline. Swap the AOI or the hazard dataset — the rest adapts.

## 1. Ingest the seismic GeoTIFF

The OpenQuake hazard dataset is distributed as a ZIP file containing a GeoTIFF. The `openquake_explorer` UDF downloads it once, extracts the `.tif`, and writes it to S3 so every subsequent UDF can read it directly without re-downloading.

```python showLineNumbers
@fused.udf
def udf(save_to_s3: bool = True):
    import pandas as pd
    import requests
    import io
    import zipfile

    url = "https://cloud.openquake.org/public.php/dav/files/6SnFk2f92JEr76H"

    @fused.cache
    def fetch_zip(url):
        response = requests.get(url, timeout=60)
        response.raise_for_status()
        return response.content

    content = fetch_zip(url)
    zf = zipfile.ZipFile(io.BytesIO(content))

    tif_name = "v2023_1_pga_475_rock_3min.tif"
    s3_path = f"s3://fused-asset/demos/seismic_data/{tif_name}"

    tif_bytes = zf.read(tif_name)

    if save_to_s3:
        import s3fs
        fs = s3fs.S3FileSystem()
        with fs.open(s3_path, "wb") as f:
            f.write(tif_bytes)

    return pd.DataFrame([{
        "filename": tif_name,
        "size_bytes": len(tif_bytes),
        "s3_path": s3_path,
        "status": "saved" if save_to_s3 else "dry_run"
    }])
```

:::tip
`@fused.cache` on `fetch_zip` means the network request only happens once. Subsequent runs read from the local cache. See the [caching guide](/guide/working-with-udfs/udf-best-practices/caching) for more.
:::

Once this UDF has been run once with `save_to_s3=True`, you can set it to `False` for all subsequent runs and skip the download entirely.

## 2. Load the seismic raster

`loading_seismic_data` reads the GeoTIFF from S3 for a given bounding box and returns a 2D NumPy array of hazard values. It uses `read_tiff_safe` from [Fused's common utilities](https://github.com/fusedio/udfs/tree/main/public/common) — a helper that handles CRS reprojection, resampling, and safe pixel-count limits automatically.

```python showLineNumbers
@fused.udf
def udf(
    bounds: fused.types.Bounds = [6.6272, 36.6197, 18.4802, 47.0921],
    chip_len: int = 512
):
    path = 's3://fused-asset/demos/seismic_data/v2023_1_pga_475_rock_3min.tif'
    common = fused.load('https://github.com/fusedio/udfs/tree/6750f1f/public/common/')
    arr = common.read_tiff_safe(bounds, path, chip_len, max_pixel=10 ** 8)
    return arr
```

The `bounds` parameter defaults to the Italy bounding box. When this UDF is used as a dependency inside `hexagonified_seismic_hazard`, bounds flow in automatically from the parent.

## 3. Raster → H3 hexagons

This is the first "aha moment" in the canvas. Most people think of rasters and hexagons as separate worlds. `hexagonified_seismic_hazard` bridges them: it takes the pixel array from step 2 and converts it into a DataFrame where each row is an H3 hex cell with an average hazard score.

```python showLineNumbers
@fused.udf
def udf(
    bounds: fused.types.Bounds = [6.6272, 36.6197, 18.4802, 47.0921],
    chip_len: int = 512,
    h3_res: int = 6
):
    import numpy as np
    common = fused.load('https://github.com/fusedio/udfs/tree/5610abb/public/common/')
    parent_udf = fused.load('loading_seimic_data')

    arr = parent_udf(bounds=bounds, chip_len=chip_len)

    # Squeeze band dimension if present
    if arr.ndim == 3:
        arr = arr[0]

    # Convert raster array → H3 hexagons
    df = common.arr_to_h3(arr, bounds, res=h3_res)

    # Collapse list of pixel values per hex to a single mean
    df["sesimic_hazard"] = df["agg_data"].apply(lambda x: float(np.mean(x)))
    df = df[["hex", "sesimic_hazard"]]
    return df
```

`arr_to_h3` samples each pixel's geographic coordinate, assigns it to an H3 cell at the chosen resolution, and aggregates all pixels that land in the same cell. The result is a flat table: one row per hex, one hazard column. See [converting data to H3](/guide/h3-analytics/converting) for more on this pattern.

:::tip H3 resolution
Resolution 6 cells are roughly 36 km² each — good for a country-scale risk overview. Resolution 7 (about 5 km²) gives finer spatial detail at the cost of more rows. See the [resolution guide](/guide/h3-analytics/resolution-guide) to pick the right level for your analysis.
:::

## 4. Explore Overture building partitions

Overture Maps buildings are stored as geo-partitioned GeoParquet files. Before reading buildings, `Overture_partitions` lets you visualize the partition footprints across Italy so you can identify which files cover your area of interest.

```python showLineNumbers
@fused.udf
def udf():
    import geopandas as gpd
    bounds = [6.6, 36.6, 18.5, 47.1]
    path = 's3://fused-asset/overture/2026-03-18.0/theme=buildings/type=building/part=2/_sample'
    common = fused.load('https://github.com/fusedio/udfs/tree/d4f567c/public/common/')
    base_path = path.rsplit('/', maxsplit=1)[0]
    df = common.table_to_tile(bounds, table=base_path, use_columns=None, min_zoom=7)
    return gpd.GeoDataFrame(df)
```

## 5. Buildings → H3 with DuckDB

`Overture_to_hex_single_file` reads a single Overture partition and aggregates building data per H3 cell — all inside one SQL query using DuckDB's built-in spatial and H3 functions.

```python showLineNumbers
@fused.udf
def udf(
    path: str = "s3://fused-asset/overture/2026-03-18.0/theme=buildings/type=building/part=2/36.parquet",
    res: int = 6
):
    common = fused.load("https://github.com/fusedio/udfs/tree/4dde28e/public/common/")
    con = common.duckdb_connect()
    gdf = con.sql(f"""
        SELECT
            h3_latlng_to_cell(
                ST_Y(ST_Centroid(geometry::GEOMETRY)),
                ST_X(ST_Centroid(geometry::GEOMETRY)),
                {res}
            ) AS hex,
            sum(ST_Area_Spheroid(geometry::GEOMETRY)) AS total_area_sqm,
            count(1) AS building_count
        FROM read_parquet('{path}')
        GROUP BY 1
        ORDER BY 1
    """).df()
    return gdf.reset_index(drop=True)
```

Three things happen inside this single SQL statement:

- **`h3_latlng_to_cell`** — assigns each building centroid to an H3 cell at the chosen resolution
- **`ST_Area_Spheroid`** — computes each building's area in square metres on the WGS84 ellipsoid
- **`GROUP BY hex`** — collapses all buildings in the same cell into `building_count` and `total_area_sqm`

This would normally require a Spark cluster or PostGIS instance. Here it runs inside a serverless UDF with zero infrastructure. See [DuckDB for geospatial](/guide/h3-analytics/aggregations) for more on this pattern.

## 6. Join on H3

With both datasets expressed as H3 hex tables, joining them is a two-line pandas merge — no spatial join, no topology checks, no geometry processing.

```python showLineNumbers
@fused.udf
def udf(
    path: str = "s3://fused-asset/overture/2026-03-18.0/theme=buildings/type=building/part=2/36.parquet",
    bounds: fused.types.Bounds = [6.6272, 36.6197, 18.4802, 47.0921],
    res: int = 6,
    chip_len: int = 512,
    h3_res: int = 6,
):
    import pandas as pd

    overture_udf = fused.load("Overture_to_hex_single_file")
    df_buildings = overture_udf(path=path, res=res)

    seismic_udf = fused.load("hexagonified_seismic_hazard")
    df_seismic = seismic_udf(bounds=bounds, chip_len=chip_len, h3_res=h3_res)

    # Normalize hex column to lowercase hex string (H3 canonical format)
    df_buildings["hex"] = df_buildings["hex"].apply(lambda x: format(int(x), 'x'))
    df_seismic["hex"] = df_seismic["hex"].apply(lambda x: format(int(x), 'x'))

    merged = pd.merge(df_buildings, df_seismic, on="hex", how="inner")
    return merged
```

This is the architectural centrepiece of the canvas. A seismic hazard raster and building vector polygons — two completely different data formats from two different organizations — are unified with a `JOIN ON hex`. No `ST_Intersects`, no projection gymnastics, no topology errors.

This works because H3 hexagons are a **universal join key**: any dataset that can be expressed as points or pixels can be assigned to the same hex grid, making multi-source joins trivially simple. Read more in the [joining H3 datasets](/guide/h3-analytics/joining) guide.

## 7. Interactive dashboard

The final node is a Canvas widget that turns the joined table into an interactive risk dashboard: a scatter chart of hazard score versus building count, KPI tiles for high-risk zones, and a filtered map layer.

The widget uses SQL queries directly against the UDF output using `{{join_over_buildings}}` syntax to reference upstream UDF results:

```json
{
  "type": "scatter-chart",
  "props": {
    "title": "Seismic Hazard vs. Number of Buildings",
    "sql": "SELECT sesimic_hazard AS x, building_count AS y, CASE WHEN sesimic_hazard > 70 AND building_count > 1000 THEN 'High Risk' ELSE 'Lower Risk' END AS series FROM {{join_over_buildings}}"
  }
}
```

The map layer filters to only show hexes where both hazard and building density exceed thresholds:

```json
{
  "sql": "SELECT hex, sesimic_hazard, building_count FROM {{join_over_buildings}} WHERE sesimic_hazard > 70 AND building_count > 1000"
}
```

:::tip Styling the map
You can customize how each hex layer looks by editing the `fillColor`, `opacity`, and `coverage` fields in the [Visualization tab](/workbench/udf-builder/viz-styling/). The joined layer uses a continuous color scale from yellow to dark red to reflect increasing hazard intensity.
:::

## Where Fused shines

**H3 as the universal join key.** A GeoTIFF raster and GeoParquet building polygons — two incompatible formats from different organizations — are joined with a plain `pd.merge`. Neither dataset needs to be reprojected or converted to match the other. Hexagonify both, merge on the cell ID, done. This approach scales to any two datasets that can be spatially indexed.

**Raster → hex in pure Python.** `hexagonified_seismic_hazard` calls `arr_to_h3` on a NumPy array. Most spatial analysis tools treat rasters and hexagons as separate paradigms. Here it's a single function call.

**DuckDB spatial SQL inside a serverless function.** `Overture_to_hex_single_file` assigns H3 cells, computes spheroidal area, and aggregates — all in one SQL query, on Fused infrastructure, with no cluster required.

**No ETL pipeline.** The canvas is the pipeline. Each UDF is a node; outputs wire directly to the next step. Changing the resolution propagates automatically. Swap the AOI or the Overture partition file — the rest adapts.

**Composable UDFs.** `join_over_buildings` calls `hexagonified_seismic_hazard` and `Overture_to_hex_single_file` directly via `fused.load`. Any UDF can call any other UDF. The canvas wires them together visually; the code stays clean.

## Limitations & next steps

This is a simplified risk overview, not a full engineering assessment. Some things to be aware of:

- The OpenQuake PGA values represent **rock-site** conditions. Soft soils amplify ground shaking significantly — a proper site-specific assessment would require soil data.
- Building exposure here is measured by **count and footprint area**, not structural type or age. Vulnerability depends heavily on construction material and year.
- Resolution 6 cells (≈36 km²) are large enough to mask local hotspots. Increasing to resolution 7 or 8 gives finer spatial detail — at the cost of sparser hex coverage in rural areas.

Ideas for extending this canvas:

- **Swap the AOI** — replace the Italy bounding box with Turkey, Japan, or the western United States, all of which have high-quality OpenQuake hazard maps.
- **Add vulnerability** — join a third dataset (e.g., building age or material from Overture) to weight exposure by structural fragility.
- **Run at scale** — use [`fused.submit()`](/guide/working-with-udfs/fused-submit) to run `Overture_to_hex_single_file` across all Overture partitions covering Italy in parallel and combine results.
- **Time-series risk** — OpenQuake also publishes scenario shake maps for historical events. Swap in a specific earthquake shake map to see which buildings were exposed to that event.
