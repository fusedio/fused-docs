# Geospatial Ingestion

Transform your data into cloud-optimized, geo-partitioned formats for fast access.

## Why Ingestion?

Large datasets (especially geospatial) can be slow to read. Proper ingestion:

- **Partitions data** into smaller, spatially-indexed chunks
- **Enables fast queries** by loading only relevant portions
- **Optimizes formats** for cloud access (GeoParquet, COG)

## Quick Start

```python
import fused

# Ingest a shapefile to geo-partitioned GeoParquet
job = fused.ingest(
    input="https://example.com/data.zip",
    output="fd://my-data/ingested/",
)
job.run_batch()
```

## When to Ingest

**Ingest when:**
- Dataset is large (>100MB)
- Data is in slow formats (CSV, Shapefile, GeoJSON)
- You need fast, repeated access
- Building UDFs that pan around a map

**Skip ingestion when:**
- Small files (&lt;100MB) already in Parquet
- One-time analysis
- Data already cloud-optimized
