# Getting Started

Learn the fundamentals of Fused: what UDFs are, how they work, and when to use them.

## What is Fused?

Fused is a platform for building serverless data workflows. Write Python functions called **UDFs** (User Defined Functions), run them locally or in the cloud, and share them as APIs.

## What is a UDF?

A UDF is a Python function decorated with `@fused.udf`:

```python
@fused.udf
def udf(name: str = "World"):
    return {"message": f"Hello, {name}!"}
```

UDFs can:
- Load data from S3, databases, or APIs
- Process with any Python library (pandas, geopandas, etc.)
- Return DataFrames, GeoDataFrames, images, or JSON
- Run locally or on Fused cloud

## Your First UDF

### 1. Install

```bash
pip install fused
```

### 2. Authenticate

```python
import fused
from fused.api import NotebookCredentials
credentials = NotebookCredentials()
```

### 3. Create a UDF

```python
@fused.udf
def udf():
    import pandas as pd
    return pd.DataFrame({"x": [1, 2, 3], "y": [4, 5, 6]})
```

### 4. Run It

```python
result = fused.run(udf)
print(result)
```

## Where to Run UDFs

| Environment | Best For |
|-------------|----------|
| Local (`engine="local"`) | Development, debugging |
| Remote (default) | Production, scaling |
| Workbench | Visual development |

## Key Concepts

- **`@fused.udf`** - Decorator that turns a function into a UDF
- **`fused.run()`** - Execute a UDF
- **`bounds`** - Viewport parameter for map-based UDFs
- **`@fused.cache`** - Cache expensive operations
