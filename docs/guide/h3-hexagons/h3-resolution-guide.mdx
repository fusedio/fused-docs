# H3 Resolution Guide

Choosing the right H3 resolution is critical for your analysis. Higher resolutions give more detail but more data; lower resolutions aggregate more but lose precision.

## How to Choose Resolution

### By Data Type

| Data Type | Recommended Resolution | Reason |
|-----------|----------------------|--------|
| GPS points | 9-11 | Balance detail vs noise |
| Building footprints | 11-12 | Match building size |
| Census/Demographics | 7-9 | Match tract/block size |
| Satellite imagery | Match pixel size | See table above |
| Global aggregations | 3-5 | Manageable cell count |

### By Visualization Zoom Level

| Map Zoom | Suggested H3 Res | Approx Cells Visible |
|----------|-----------------|---------------------|
| 1-3 | 0-2 | Hundreds |
| 4-6 | 3-4 | Thousands |
| 7-9 | 5-6 | Tens of thousands |
| 10-12 | 7-8 | Hundreds of thousands |
| 13-15 | 9-10 | Millions |
| 16+ | 11+ | Very high density |

### By Analysis Type

- **Aggregation/Heatmaps:** Use coarser resolution (7-9) for clear patterns
- **Precise location:** Use finer resolution (11-13)
- **Multi-scale analysis:** Ingest at fine resolution, use `h3.cell_to_parent()` to aggregate up

## Dynamic Resolution Based on Bounds

A common pattern is to automatically choose resolution based on the current map viewport:

```python
@fused.udf
def udf(bounds: fused.types.Bounds = None):
    common = fused.load("https://github.com/fusedio/udfs/tree/main/public/common/")
    
    # Automatically choose resolution based on zoom
    res = common.bounds_to_res(bounds)
    
    # Or manually map zoom to resolution
    zoom = common.estimate_zoom(bounds)
    res = max(3, min(12, int(3 + (zoom - 3) / 1.7)))
    
    return load_h3_data(bounds, res)
```
