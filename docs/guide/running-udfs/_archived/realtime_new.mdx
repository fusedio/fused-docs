# Realtime Execution

Realtime is the default way to run UDFs. Jobs execute immediately with no startup time.

**Limits:**
- Max 120 seconds execution time
- Standard memory allocation (~8GB)

---

## `fused.run()`

The simplest way to execute a UDF from Python:

```python
import fused

# By name (from your account)
result = fused.run("My_UDF_Name")

# Public UDF (prefix with UDF_)
result = fused.run("UDF_Single_point_Eiffel_Tower")

# By shared token
result = fused.run("fsh_abc123...")

# With parameters
result = fused.run("My_UDF", param1="value", bounds=[...])
```

---

## Ways to Reference a UDF

| Method | When to Use | Example |
|--------|-------------|---------|
| **Name** | Your own UDFs | `fused.run("My_UDF")` |
| **Public name** | Community UDFs | `fused.run("UDF_Overture_Maps")` |
| **Token** | Share without code | `fused.run("fsh_abc...")` |
| **Teammate** | Team collaboration | `fused.run("teammate@co.io/UDF")` |
| **Git commit** | Production stability | `fused.load("https://github.com/.../tree/{hash}/...")` |

---

## Local vs Remote Execution

```python
# Run on Fused servers (default)
fused.run(my_udf)

# Run locally
fused.run(my_udf, engine="local")
```

Use `engine="local"` for debugging or when not authenticated.

---

## HTTPS Endpoints

Every saved UDF can be called via HTTP:

```
https://www.fused.io/server/v1/realtime-shared/{token}/run/file?format=json
```

**Format options:**
- Tables: `parquet`, `geojson`, `json`, `csv`, `mvt`
- Images: `png`, `jpg`, `tiff`

See [HTTPS Endpoints](/guide/running-udfs/http-endpoints) for details.
