Fused publishes a [skills plugin](https://github.com/fusedio/skills) that teaches AI coding tools the Fused-specific formats they need to author canvases: the `canvas.toml` schema, JSON UI widget definitions, and the `fused` CLI. Once installed, the AI understands how to create and modify Fused canvases without you having to explain the format every time.

The plugin covers three skill areas:

| Skill | What it teaches |
|---|---|
| `canvas-toml` | `canvas.toml` format — nodes, edges, viewport, folder layout |
| `fused-cli` | `fused` CLI — push, run, share, and manage UDFs from the terminal |
| `json-ui-schemas` | Widget JSON schemas — text inputs, dropdowns, charts, maps, SQL tables |

## Claude Code

Claude Code supports the Fused plugin natively via its plugin marketplace.

**Option A — via the `fused` CLI:**

```bash
pip install fused
fused claude plugin add
```

**Option B — via Claude Code directly:**

```bash
claude plugin marketplace add fusedio/claude-plugins
claude plugin install fused@fused-marketplace
```

To update or remove later:

```bash
claude plugin update fused@fused-marketplace
claude plugin remove fused
```

Once installed, Claude Code automatically applies the Fused skills whenever you work on canvas files.

## Codex

OpenAI Codex CLI uses `AGENTS.md` files to provide persistent context, similar to how Claude Code uses `CLAUDE.md`. There is no plugin marketplace, so paste the skill content directly.

1. Download the skill files from [github.com/fusedio/skills](https://github.com/fusedio/skills/tree/main/fused-plugin/skills) — copy `canvas-toml`, `fused-cli`, and `json-ui-schemas`.
2. Paste the relevant content into your project's `AGENTS.md` file.
3. Codex will apply that context automatically on every run in that directory.

## Cursor

Cursor uses **Rules** to inject persistent context into every chat. There is no plugin install — create a rules file and paste the skill content into it.

**Project-level (recommended):** create a `.cursor/rules/fused.md` file in your project root and paste the skill content from [github.com/fusedio/skills](https://github.com/fusedio/skills/tree/main/fused-plugin/skills). This file is version-controlled and scoped to your repo.

**Global:** go to **Cursor Settings → Rules for AI** and paste the content there to apply it across all projects.

Once set, Cursor's Agent mode will understand Fused canvas structure without additional prompting.

## Windsurf

Windsurf uses **Rules** files to inject persistent context into Cascade. There is no plugin install.

**Project-level (recommended):** create a `.windsurf/rules/fused.md` file in your project root. Add frontmatter to control when it activates, then paste the skill content from [github.com/fusedio/skills](https://github.com/fusedio/skills/tree/main/fused-plugin/skills):

```markdown
---
trigger: always_on
---

<paste skill content here>
```

**Global:** add the skill content to `~/.codeium/windsurf/memories/global_rules.md` to apply it across all projects.

Windsurf's Cascade mode will pick up the rules automatically.
