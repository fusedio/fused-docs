Fused publishes a [skills plugin](https://github.com/fusedio/skills) that teaches AI coding tools the Fused-specific formats they need to author canvases: the `canvas.toml` schema, JSON UI widget definitions, and the `fused` CLI. Once installed, the AI understands how to create and modify Fused canvases without you having to explain the format every time.

The plugin covers three skill areas:

| Skill | What it teaches |
|---|---|
| `canvas-toml` | `canvas.toml` format — nodes, edges, viewport, folder layout |
| `fused-cli` | `fused` CLI — push, run, share, and manage UDFs from the terminal |
| `json-ui-schemas` | Widget JSON schemas — text inputs, dropdowns, charts, maps, SQL tables |

---

## Claude Code

Claude Code supports the Fused plugin natively via its plugin marketplace.

**Option A — via the `fused` CLI:**

```bash
pip install fused
fused claude plugin add
```

**Option B — via Claude Code directly:**

```bash
claude plugin marketplace add fusedio/claude-plugins
claude plugin install fused@fused-marketplace
```

To update or remove later:

```bash
claude plugin update fused@fused-marketplace
claude plugin remove fused
```

Once installed, Claude Code automatically applies the Fused skills whenever you work on canvas files.

## Codex

The Fused plugin is not yet available in the Codex skill marketplace. In the meantime, install the skills directly from GitHub using Codex's built-in installer — ask Codex:

```
Install skills from github.com/fusedio/skills
```

This places the `canvas-toml`, `fused-cli`, and `json-ui-schemas` skill directories into `~/.codex/skills/`. Restart Codex after installation for the skills to be picked up.

---

## Cursor

The Fused plugin is not yet available in the Cursor Marketplace. In the meantime, copy the skill directories from [github.com/fusedio/skills](https://github.com/fusedio/skills/tree/main/fused-plugin/skills) into `.cursor/skills/` in your project root (e.g. `.cursor/skills/canvas-toml/SKILL.md`). Cursor will discover and apply them automatically in Agent mode.
