# job2.ai — LLM Integration

Use `job2.ai` to call large language models directly inside Fused UDFs — no API keys to pass, no tokens to manage. It automatically uses your Fused account's purchased token quota.

## Import

```python
from job2 import ai
from job2.ai import AiModel
```

`job2` is available in the Fused runtime. No installation required.

## Quickstart

```python
@fused.udf
def udf():
    from job2 import ai
    from job2.ai import AiModel

    response = ai.run(
        "Summarize the key benefits of vector databases.",
        system_prompt="You are a concise technical writer.",
        model=AiModel.GPT_OSS_120B,
    )
    return response.text
```

## `ai.run()`

Send a prompt to an LLM and get a response.

```python
response = ai.run(
    prompt,
    system_prompt=SYSTEM_PROMPT,
    model=AiModel.GPT_OSS_120B,
    temperature=0.3,
)
result_text = response.text
```

### Parameters

| Parameter | Type | Description |
| --- | --- | --- |
| `prompt` | `str` | The user message to send to the model |
| `system_prompt` | `str` | Instructions that define the model's behavior and persona |
| `model` | `AiModel \| str` | The model to use (see [Models](#models) below) |
| `temperature` | `float` | Sampling temperature. Lower = more deterministic. Default: `0.3` |
| `tools` | `list` | Optional list of tool definitions in OpenAI function-calling format |
| `tool_handler` | `callable` | Optional handler `fn(name, params) -> str` invoked when the model calls a tool |
| `max_iterations` | `int` | Max agentic tool-calling rounds when using tools. Default: `8` |

### Returns

A response object with a `.text` attribute containing the model's reply as a string.

## Models

Use the `AiModel` enum or pass the model name as a string:

```python
from job2.ai import AiModel

AiModel.GPT_OSS_120B   # default, high-quality open-source model
```

## System prompts

System prompts define the model's persona and constraints:

```python
SYSTEM_PROMPT = """You are a helpful data assistant. Be concise and friendly.
Only answer questions based on the data provided — do not use general knowledge."""

response = ai.run(prompt, system_prompt=SYSTEM_PROMPT, model=AiModel.GPT_OSS_120B)
```

## Tool calling (agentic mode)

`ai.run()` supports OpenAI-style function calling. Define tools as JSON schemas and handle calls with a `tool_handler`:

```python
tools = [{
    "type": "function",
    "function": {
        "name": "get_weather",
        "description": "Get the current weather for a city.",
        "parameters": {
            "type": "object",
            "properties": {
                "city": {"type": "string"}
            },
            "required": ["city"],
        },
    },
}]

def handle_tool(name, params):
    if name == "get_weather":
        return f"Sunny, 72°F in {params['city']}"

response = ai.run(
    "What's the weather in Austin?",
    system_prompt="You are a weather assistant.",
    tools=tools,
    tool_handler=handle_tool,
    model=AiModel.GPT_OSS_120B,
)
```

This is the same mechanism that powers the [AI Chat's user-defined tools](/guide/data-input-outputs/import-connection/ai-data-connection) — but called programmatically from your UDF instead of through the Workbench UI. You can use it to build [agent-style UDFs](/guide/working-with-udfs/udf-best-practices/agents) that chain LLM reasoning with data tools.

## Token usage

:::warning
Every call to `ai.run()` counts against your Fused account's AI token quota.
:::

Use [`@fused.cache`](/python-sdk/api-reference/fused-cache) to avoid re-running identical prompts during development:

```python
@fused.cache
def cached_llm_call(text, system_prompt):
    from job2 import ai
    from job2.ai import AiModel
    response = ai.run(text, system_prompt=system_prompt, model=AiModel.GPT_OSS_120B)
    return response.text
```

## Notes

- No API key configuration required — authentication is handled automatically via your Fused session.
- `job2.ai` is only available within the Fused runtime (UDFs and the Fused API). It cannot be used outside of Fused.
- For parallel LLM calls across many rows, combine with `common.run_pool()` from the [Fused common utilities](https://github.com/fusedio/udfs/tree/main/public/common/).
