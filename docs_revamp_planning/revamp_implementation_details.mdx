## Redirect Logic

### How Redirects Work

We use `@docusaurus/plugin-client-redirects` in `docusaurus.config.ts`. Add entries to the `redirects` array:

```typescript
{
  to: "/new/path",      // Where users should go
  from: ["/old/path"],  // Old URL(s) that redirect
}
```

**Note:** Anchors (`#section`) are NOT preserved - users land on the new page but won't auto-scroll to the section.

### To Enable Redirects

1. Delete the old doc folders (or move them to an `_archive/` folder):
   ```bash
   rm -rf docs/tutorials docs/core-concepts docs/workbench docs/python-sdk
   ```

2. Uncomment the redirects in `docusaurus.config.ts` (search for "DOCS REVAMP REDIRECTS")

3. Build and test:
   ```bash
   npm run build && npm run serve
   ```

### Testing Redirects

```bash
# Build and serve locally
npm run build
npm run serve

# Test a redirect (should return 200 after redirect)
curl -L -o /dev/null -w "%{http_code}" http://localhost:3000/tutorials/Analytics%20&%20Dashboard/standalone-maps
```
