Enregistrez des constantes dans un fichier `.env` pour les rendre disponibles aux UDF en tant que variables d'environnement. Vous devriez utiliser le [gestionnaire de secrets](/workbench/preferences/#secrets-management) pour des informations sensibles comme les clés API.

Tout d'abord, exécutez un [File UDF](/tutorials/Geospatial%20with%20Fused/filetile/#single-udf) qui définit des variables dans un fichier `.env` dans le [`/mnt/cache/` directory](/core-concepts/content-management/file-system/#mntcache-disk).

```py
@fused.udf
def udf():
    env_vars = """
    MY_ENV_VAR=123
    """

    # Path to .env file in disk file system
    env_file_path = '/mnt/cache/.env'

    # Write the environment variables to the .env file
    with open(env_file_path, 'w') as file:
        file.write(env_vars)
```

Maintenant, tout UDF peut charger les valeurs depuis `.env` en tant que variables d'environnement avec [`load_dotenv`](https://pypi.org/project/python-dotenv/) et y accéder avec [os.getenv](https://www.geeksforgeeks.org/python-os-getenv-method/).

```py
@fused.udf
def udf():
    from dotenv import load_dotenv

    # Load environment variable
    env_file_path = '/mnt/cache/.env'
    load_dotenv(env_file_path, override=True)

    # Access environment variable
    print(f"Updated MY_ENV_VAR: {os.getenv('MY_ENV_VAR')}")
```
