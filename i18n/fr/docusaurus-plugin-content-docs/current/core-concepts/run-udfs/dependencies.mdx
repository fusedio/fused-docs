# Dépendances

Pour garder les choses simples, Fused maintient une seule image d'exécution. Cela signifie que tout UDF que vous [exécutez](/core-concepts/run-udfs/) sera exécuté avec ces dépendances par défaut.

## Dépendances UDF

Les packages Python sont listés ci-dessous et peuvent également être trouvés dans [ce](https://github.com/fusedio/fused-docs/blob/main/static/requirements_fused.txt) fichier `.txt` public.

import txt from 'raw-loader!/requirements_fused.txt';

<details>
    <summary>Packages Python de dépendance UDF</summary>
    <pre>
      <code>{`${txt}`}</code>
    </pre>
</details>

## [BETA] Installez vos propres dépendances

Le moyen le plus simple d'ajouter votre propre bibliothèque est d'exécuter l'application dédiée [Package Management Fused](https://www.fused.io/workbench/apps/catalog/Package_Management-688daddc-dd51-4173-9003-d2bb5ddee72d). Cette application vous permet de créer un environnement différent et d'ajouter n'importe quel module que vous souhaitez.

:::note
Vous devez vous assurer que vous avez accès aux [Fused Apps](/workbench/app-builder/app-overview/) pour pouvoir exécuter cela.
:::

![Application de gestion de package bêta](/img/core-concepts/run-udfs/beta_package_management_app.png)

Vous devrez ensuite importer le chemin de l'environnement dans votre UDF :

```python showLineNumbers
@fused.udf
def udf():
  import sys;
  sys.path.append(f"/mount/envs/demo_env/lib/python3.11/site-packages/")

  # Logique utilisant votre package dédié
  return
```
