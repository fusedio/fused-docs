### Erreurs courantes

<details>
<summary>Erreur : `Access is not configured for you in the Fused Workbench. Please refresh the page if you think this is an error, or get in touch if you require further help. Cause: Realtime instance not configured.`</summary>

Cette erreur se produit lorsque vous essayez d'exécuter un UDF avec un compte associé à un environnement de workspace qui n'a pas d'instance en temps réel configurée. Cela signifie qu'il n'y a pas de nœuds de travail disponibles pour exécuter l'UDF. Pour résoudre ce problème, veuillez contacter l'équipe Fused pour vous assurer que votre compte est associé à un environnement avec une instance en temps réel.

Lors du dépannage de cette erreur, il peut être utile de naviguer vers la page de profil utilisateur de votre compte pour déterminer si le compte est associé à un environnement et une instance en temps réel, comme montré ici.

![user settings](/img/user_settings.png)

</details>



<details>
<summary>Erreur : `No such file or directory: '/mnt/cache/'`</summary>

Cette erreur se produit lorsqu'un UDF tente d'accéder au [`/mnt/cache` disk](/core-concepts/content-management/file-system/#mntcache-disk) lorsqu'il n'est pas disponible pour l'environnement. Pour résoudre ce problème, veuillez contacter l'équipe Fused pour vous assurer que le répertoire de cache est disponible pour votre compte.
</details>

<details>
<summary>Erreur : `No space left on the device: '/tmp/'`</summary>

Cette erreur se produit lorsqu'un UDF tente d'écrire plus de données que le répertoire `/tmp` de l'instance en temps réel peut gérer. Les instances en temps réel ont une quantité limitée d'espace disponible et sont éphémères entre les exécutions. Vous voudrez peut-être envisager d'écrire sur le [`/mnt/cache` disk](/core-concepts/content-management/file-system/#mntcache-disk) à la place.
</details>

<details>
<summary>Erreur : `Quota limit: Number of running instances`</summary>

Les travaux par lots de Fused, qui sont initiés avec [run_batch](/python-sdk/top-level-functions/#jobrun_batch), nécessitent qu'un quota de serveur soit activé pour votre compte. Cela inclut les travaux d'[ingestion](/tutorials/Geospatial%20with%20Fused/geospatial-data-ingestion/) de données. Si vous rencontrez cette erreur, veuillez contacter l'équipe Fused pour demander une augmentation du quota attribué à votre compte.
</details>

<details>
<summary>Erreur : `Application error: a client-side exception has occurred (see browser console for more information).`</summary>

Dans le cas où vous rencontrez cette erreur, veuillez réinitialiser le cache et les cookies de votre navigateur. Si l'erreur persiste, veuillez contacter l'équipe Fused pour obtenir une assistance supplémentaire.
</details>


<details>
<summary>Erreur : `MY_UDF... not saved because a UDF or app wit that name already exists.`</summary>

Il est possible que vous soyez en train de modifier un UDF "local" qui a le même nom qu'un UDF d'équipe. Pour résoudre ce problème, vous pouvez soit renommer l'UDF, soit, si votre intention est de mettre à jour l'UDF d'équipe, [le pousser vers GitHub](/workbench/udf-catalog/#publish-a-udf-to-a-github-repository).

</details>


<details>
<summary>Erreur : `Failed to create share token: {"detail":"UDF not found"}`</summary>

Pour résoudre cette erreur, enregistrez à nouveau l'UDF et actualisez votre fenêtre de navigateur. Si le problème persiste, renommez l'UDF et réessayez.

</details>

<details>
<summary>Erreur : `ModuleNotFoundError: No module named 'geopandas' / 'xarray'`</summary>

Cette erreur se produit lorsque votre UDF retourne un `GeoDataFrame` ou un Dataset `xarray` mais que votre environnement local n'a pas le package `geopandas` ou `xarray` installé pour le manipuler.

Le package autonome `fused` n'a pas `geopandas` ou `xarray` ou d'autres dépendances optionnelles installées pour le garder léger.

Cela peut être corrigé de 2 manières :

1. Lors de l'exécution de fused localement

Vous pouvez [installer](/python-sdk/#python-install) `fused` avec toutes les dépendances optionnelles :

```bash
pip install "fused[all]"
```

2. Lors de l'exécution d'un UDF dans une [Fused App](/workbench/app-builder/)

Installez le package en utilisant `micropip` en haut de votre application :

```python
import micropip
await micropip.install(['geopandas'])
# await micropip.install(['xarray']) # si vous manquiez xarray
import geopandas as gpd
```

Vous pouvez installer plusieurs packages de la même manière :
```python
await micropip.install(['geopandas', 'xarray'])
```

</details>


<details>
<summary>L'utilisation de la mémoire de Workbench me dit "Inconnu"</summary>

Nous faisons de notre mieux pour récupérer l'utilisation de la mémoire pour les UDF que vous exécutez dans Workbench afin de vous aider à surveiller et optimiser vos UDF.

Il y a plusieurs raisons pour lesquelles nous pourrions ne pas être en mesure de récupérer l'utilisation de la mémoire :
1. **Exécution d'un token partagé, par exemple un UDF public** : Dans ce cas, nous ne partageons pas l'utilisation de la mémoire car le constructeur original de l'UDF pourrait ne pas vouloir révéler le fonctionnement interne de son UDF.
2. **Exécution d'un [UDF mis en cache](/core-concepts/cache/)** : Si vous exécutez un UDF qui a été mis en cache (réexécution du même UDF avec les mêmes paramètres), aucune mémoire n'a été utilisée.

</details>
