## `run_tile`

```python showLineNumbers
def run_tile(email: str,
             id: Optional[str] = None,
             *,
             x: int,
             y: int,
             z: int,
             _dtype_out_vector: str = DEFAULT_DTYPE_VECTOR,
             _dtype_out_raster: str = DEFAULT_DTYPE_RASTER,
             _client_id: Optional[str] = None,
             **params) -> Optional[Union["pd.DataFrame", "xr.Dataset"]]
```

Exécute une UDF privée basée sur des tuiles indexées sous l'email et l'ID spécifiés. L'utilisateur appelant doit avoir les autorisations nécessaires pour exécuter la UDF.

Cette fonction construit une URL pour exécuter une UDF sur une tuile spécifique définie par ses coordonnées x, y et z, et
envoie une requête au serveur. Elle prend en charge la personnalisation des types de données de sortie pour les données vectorielles et raster,
ainsi que des paramètres supplémentaires pour l'exécution de la UDF.

**Arguments**:

- `email` _str_ - Adresse e-mail du compte utilisateur associé à la UDF.
- `id` _Optional[str]_ - Identifiant unique pour la UDF. Si None, l'e-mail de l'utilisateur est utilisé comme ID.
- `x` _int_ - La coordonnée x de la tuile.
- `y` _int_ - La coordonnée y de la tuile.
- `z` _int_ - Le niveau de zoom de la tuile.
- `_dtype_out_vector` _str_ - Type de données souhaité pour la sortie vectorielle. Par défaut, un type prédéfini est utilisé.
- `_dtype_out_raster` _str_ - Type de données souhaité pour la sortie raster. Par défaut, un type prédéfini est utilisé.
- `_client_id` _Optional[str]_ - Identifiant du client pour l'utilisation de l'API. Si None, un ID client par défaut ou global peut être utilisé.
- `**params` - Arguments supplémentaires pour l'exécution de la UDF.


**Returns**:

  La réponse du serveur après l'exécution de la UDF sur la tuile spécifiée.

## `run_shared_file`

```python showLineNumbers
def run_shared_file(token: str,
                    *,
                    _dtype_out_vector: str = DEFAULT_DTYPE_VECTOR,
                    _dtype_out_raster: str = DEFAULT_DTYPE_RASTER,
                    **params) -> Optional[Union["pd.DataFrame", "xr.Dataset"]]
```

Exécute une UDF partagée basée sur des fichiers.

Cette fonction construit une URL pour exécuter une opération sur un fichier accessible via un jeton partagé. Elle permet la personnalisation des types de données de sortie pour les données vectorielles et raster et prend en charge des paramètres supplémentaires pour l'exécution de l'opération.

**Arguments**:

- `token` _str_ - Un jeton d'accès partagé qui autorise l'opération.
- `_dtype_out_vector` _str_ - Type de données souhaité pour la sortie vectorielle, par défaut un type prédéfini.
- `_dtype_out_raster` _str_ - Type de données souhaité pour la sortie raster, par défaut un type prédéfini.
- `**params` - Arguments supplémentaires pour l'exécution de l'opération.


**Returns**:

  La réponse du serveur après l'exécution de l'opération sur le fichier.


**Raises**:

- `Exception` - Décrit diverses exceptions qui pourraient survenir lors de l'exécution de la fonction, y compris mais sans s'y limiter, des paramètres invalides, des erreurs réseau, des erreurs d'accès non autorisé ou des erreurs côté serveur.


:::note

Cette fonction est conçue pour accéder à des opérations partagées qui nécessitent un jeton pour l'autorisation. Elle nécessite un accès réseau pour communiquer avec le serveur hébergeant ces opérations et peut entraîner des coûts ou des retards de transmission de données en fonction des performances du réseau.

:::

## `run_tile_async`

```python showLineNumbers
async def run_tile_async(
        email: str,
        id: Optional[str] = None,
        *,

        y: int,
        z: int,
        _dtype_out_vector: str = DEFAULT_DTYPE_VECTOR,
        _dtype_out_raster: str = DEFAULT_DTYPE_RASTER,
        _client_id: Optional[str] = None,
        **params) -> Optional[Union["pd.DataFrame", "xr.Dataset"]]
```

Exécute de manière asynchrone une UDF privée basée sur des tuiles indexées sous l'email et l'ID spécifiés. L'utilisateur appelant doit avoir les autorisations nécessaires pour exécuter la UDF.

Cette fonction construit une URL pour exécuter de manière asynchrone une UDF sur une tuile spécifique définie par ses coordonnées x, y et z. Elle prend en charge la personnalisation des types de données de sortie pour les données vectorielles et raster, et accepte des paramètres supplémentaires pour l'exécution de la UDF.

**Arguments**:

- `email` _str_ - Adresse e-mail de l'utilisateur. Utilisée pour identifier les UDF sauvegardées de l'utilisateur. Si l'ID n'est pas fourni, l'e-mail est également utilisé comme ID.
- `id` _Optional[str]_ - Identifiant unique pour la UDF. Si None, l'e-mail de l'utilisateur est utilisé comme ID.
- `x` _int_ - La coordonnée x de la tuile.
- `y` _int_ - La coordonnée y de la tuile.
- `z` _int_ - Le niveau de zoom de la tuile.
- `_dtype_out_vector` _str_ - Type de données souhaité pour la sortie vectorielle. Par défaut, un type prédéfini est utilisé.
- `_dtype_out_raster` _str_ - Type de données souhaité pour la sortie raster. Par défaut, un type prédéfini est utilisé.
- `_client_id` _Optional[str]_ - Identifiant du client pour l'utilisation de l'API. Si None, un ID client par défaut ou global peut être utilisé.
- `**params` - Arguments supplémentaires pour l'exécution de la UDF.


**Returns**:

  Une coroutine qui, lorsqu'elle est attendue, envoie une requête au serveur pour exécuter la UDF sur la tuile spécifiée et retourne la réponse du serveur. Le format et le contenu de la réponse dépendent de l'implémentation de la UDF et du format de réponse du serveur.
