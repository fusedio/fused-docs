## fused.options

Liste des options de configuration globales.

Cet objet contient un ensemble d'options de configuration qui contrôlent le comportement global de la bibliothèque. Cet objet peut être utilisé pour modifier les options.

**Exemples :**

Changez l'option `request_timeout` de sa valeur par défaut à 60 secondes :

```py
fused.options.request_timeout = 60
```


## Options

### __dir__

```python
__dir__() -> List[str]
```

### base_url

```python
base_url: str = PROD_DEFAULT_BASE_URL
```

Point de terminaison de l'API Fused

### auth

```python
auth: AuthOptions = Field(default_factory=AuthOptions)
```

Options pour l'authentification.

### show

```python
show: ShowOptions = Field(default_factory=ShowOptions)
```

Options pour les représentations d'objet et comment les données sont affichées pour le débogage.

### max_workers

```python
max_workers: int = 16
```

Nombre maximum de threads, lors de l'exécution de requêtes multithread.

### run_timeout

```python
run_timeout: float = 130
```

Délai d'expiration des requêtes pour les requêtes d'exécution UDF au service Fused.

### request_timeout

```python
request_timeout: Union[Tuple[float, float], float, None] = 5
```

Délai d'expiration des requêtes pour le service Fused.

Peut être défini sur un tuple de délai d'expiration de connexion et de délai d'expiration de lecture.

### request_max_retries

```python
request_max_retries: int = 5
```

Nombre maximum de tentatives pour les requêtes API.

### request_retry_base_delay

```python
request_retry_base_delay: float = 1.0
```

Délai de base avant de réessayer une requête API en secondes.

### realtime_client_id

```python
realtime_client_id: Optional[StrictStr] = None
```

ID client pour le service en temps réel.

### max_recursion_factor

```python
max_recursion_factor: int = 5
```

Facteur de récursion maximum pour les UDF. Cela est utilisé pour limiter le nombre d'appels récursifs aux UDF. Si un UDF dépasse cette limite, une erreur sera levée.

### save_user_settings

```python
save_user_settings: StrictBool = True
```

Enregistrer les paramètres par utilisateur tels que les identifiants et les ID d'environnement.

### default_udf_run_engine

```python
default_udf_run_engine: Optional[StrictStr] = None
```

Moteur par défaut pour exécuter les UDF, l'un des suivants : "local" ou "remote".

### default_validate_imports

```python
default_validate_imports: StrictBool = False
```

Par défaut, pour savoir s'il faut valider les imports dans les UDF avant `run_local`,
`run_batch`.

### prompt_to_login

```python
prompt_to_login: StrictBool = False
```

Inviter automatiquement l'utilisateur à se connecter lors de l'importation de Fused.

### no_login

```python
no_login: StrictBool = False
```

S'il est défini, Fused ne tentera pas de se connecter automatiquement lorsque cela est nécessaire.

### pyodide_async_requests

```python
pyodide_async_requests: StrictBool = False
```

S'il est défini, Fused est appelé à l'intérieur de Pyodide et doit utiliser Pyodide
pour les requêtes HTTP asynchrones.

### cache_directory

```python
cache_directory: Path | None = None
```

Le répertoire de base pour stocker les résultats mis en cache.

### data_directory

```python
data_directory: Path = None
```

Le répertoire de base pour stocker les résultats de données. Remarque : si le type de stockage est 'object', alors ce chemin est relatif à
fd_prefix.

### temp_directory

```python
temp_directory: Path = Path(tempfile.gettempdir())
```

Le répertoire de base pour stocker les fichiers temporaires.

### never_import

```python
never_import: StrictBool = False
```

Ne jamais importer le code UDF lors du chargement des UDF.

### gcs_secret

```python
gcs_secret: str = 'gcs_fused'
```

Nom secret pour les identifiants GCS.

### gcs_filename

```python
gcs_filename: str = '/tmp/.gcs.fused'
```

Nom de fichier pour enregistrer les identifiants GCS temporaires localement ou dans l'instance rt2.

### gcp_project_name

```python
gcp_project_name: Optional[StrictStr] = None
```

Nom du projet pour GCS à utiliser pour les opérations GCS.

### logging

```python
logging: StrictBool = Field(default=False, validate_default=True)
```

Contrôler la journalisation pour Fused.

### verbose_udf_runs

```python
verbose_udf_runs: StrictBool = True
```

Savoir s'il faut imprimer les journaux des exécutions UDF par défaut.

### default_run_headers

```python
default_run_headers: Optional[Dict[str, str]] = {
    "X-Fused-Cache-Disable": "true"
}

```

(Avancé) En-têtes par défaut à inclure avec les requêtes d'exécution UDF.

### default_dtype_out_vector

```python
default_dtype_out_vector: StrictStr = 'parquet'
```

Type de transfert par défaut pour les données vectorielles (tabulaires).

### default_dtype_out_raster

```python
default_dtype_out_raster: StrictStr = 'npy,tiff'
```

Type de transfert par défaut pour les données raster.

### fd_prefix

```python
fd_prefix: Optional[str] = None
```

S'il est défini, où les URL du schéma fd:// seront résolues. Par défaut, cela sera inféré de votre compte utilisateur.

### verbose_cached_functions

```python
verbose_cached_functions: StrictBool = True
```

Savoir s'il faut imprimer les journaux des fonctions décorées de cache par défaut.

### local_engine_cache

```python
local_engine_cache: StrictBool = True
```

Activer le cache UDF avec le moteur local.

### default_send_status_email

```python
default_send_status_email: StrictBool = True
```

Savoir s'il faut envoyer un e-mail de statut à l'utilisateur lorsqu'un travail est terminé.

### cache_storage

```python
cache_storage: StorageStr = 'auto'
```

Spécifiez le type de stockage de cache par défaut.

### base_web_url

```python
base_web_url
```

### save

```python
save()
```

Enregistrer les options Fused dans `~/.fused/settings.toml`. Elles seront automatiquement
rechargées la prochaine fois que fused-py sera importé.
