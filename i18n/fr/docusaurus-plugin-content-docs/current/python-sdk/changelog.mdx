# Journal des modifications

## v1.25.1 (2025-10-29)

**Nouvelles fonctionnalités :**
- Relations expérimentales (arêtes) entre les nœuds du canevas, permettant de créer une nouvelle UDF à partir d'une UDF de données existante, de voir les relations de données entre les UDF, et de déclencher une chaîne d'UDF.

**Améliorations :**
- Les nœuds du canevas afficheront "Rendu" lorsqu'ils attendent le navigateur.
- Le chat AI peut maintenant voir la liste des fonctions et des UDF.
- Le chat AI peut appeler un outil pour exécuter l'UDF lui-même.
- Le chat AI dispose maintenant de modèles exacto.
- Le Workbench demandera une confirmation si vous quittez l'onglet pendant qu'un enregistrement est en cours.
- Le catalogue UDF dans le canevas inclut maintenant un bouton pour ouvrir l'URL partagée.
- Le repliement de code est maintenant configuré pour ne pas replier la fonction `udf`.
- Des noms UDF aléatoires sont choisis à partir d'un plus grand pool.
- Amélioration des performances de chargement des points de terminaison UDF.
- La sortie HTML utilisée dans le Canevas sera par défaut complétée par un code et un style supplémentaires.
- `Xarray` mis à jour vers 2025.10.01.
- Les paramètres passés à `fused.run` seront plus régulièrement sérialisés.

**Corrections de bugs :**
- Ajustement de l'interface utilisateur pour le canevas en pause.
- Les actions UDF dans le canevas auront le même aspect que le reste de l'application.
- Ajustement de l'emplacement du lien Hyparquet.
- Correction de la suppression des répertoires S3 avec de nombreux objets.
- Correction de l'enregistrement de tous les UDF lorsque les noms sont en conflit.
- Les UDF favoris peuvent être enregistrés dans le cloud.
- Le chat AI aura un ordre plus cohérent des éléments de contexte.
- Le modèle AI sélectionné pour les modifications en ligne sera conservé.
- Correction de bugs avec les UDF affichés comme "en cours d'exécution" dans le canevas alors qu'ils ne l'étaient pas réellement.
- Correction de bugs où la liste du catalogue UDF serait tronquée pour les utilisateurs avec de nombreux UDF.
- Le canevas ne réorganisera pas les nœuds en mode partagé.
- Correction d'une erreur lors de l'exécution d'UDF avec de nombreuses lignes de code et l'option profiler activée.
- Correction de la désactivation de "supprimer" sur certains UDF dans le canevas.
- La suppression de notes autocollantes dans le canevas entraînera leur suppression permanente.
- Les UDF supprimés des paramètres seront également supprimés du Workbench.
- Correction d'un bug où les UDF appelés dans un UDF appelé à partir d'un jeton partagé ne pourraient pas eux-mêmes appeler des UDF par nom.
- Correction du message d'erreur lorsqu'une valeur `cache_max_age` invalide était passée.

## v1.25.0 (2025-10-22)

**Nouvelles fonctionnalités :**
- Le Workbench analyse maintenant automatiquement les messages d'erreur des UDF et suggère une correction AI.
- Ajout de l'API `fused.context` pour accéder aux parties avancées de la requête UDF.
- Ajout du support du modèle Claude Haiku.
- Les UDF peuvent maintenant être favoris dans le Workbench.

**Améliorations :**
- [Jetons partagés](/core-concepts/run-udfs/run-small-udfs/#shared-token) utiliseront désormais `udf.ai`.
- Les URL de [jetons partagés](/core-concepts/run-udfs/run-small-udfs/#shared-token) par défaut seront en HTML lorsque aucun format n'est spécifié.
- La liste des UDF dans le canevas affiche le statut de l'UDF.
- Le panneau des résultats ne s'affichera plus dans le canevas.
- Le [catalogue UDF](https://www.fused.io/workbench/catalog) du Workbench s'ouvrira maintenant plus rapidement avec des UDF mis en cache.
- `fused.run` sérialisera plus de valeurs à passer aux UDF.
- Le canevas a une nouvelle fonctionnalité d'auto-organisation.
- Les pages du Workbench telles que la liste des travaux et l'explorateur de fichiers sont maintenant des superpositions modales.
- Le bouton Exécuter/Arrêter a été déplacé du panneau des résultats vers le panneau du Canevas.
- Ajustement de la façon dont l'AI termine les réponses, ajout de nouveaux résumés et auto-nommage.
- L'index du jeu de données inclut maintenant des statistiques de colonne.
- La vue en tableau du Workbench affiche maintenant le nombre d'enregistrements.
- Le canevas prend désormais en charge les messages de type Paramètre envoyés vers/depuis les UDF.
- Le bouton *Réinitialiser* dans le Workbench réinitialise maintenant les paramètres définis sur les UDF.
- `fused.cache` ne prend plus en compte les variables (potentiellement modifiées) définies en dehors du corps de la fonction.
- Amélioration pour la mise en cache des "UDF imbriqués" (un udf défini dans le code et exécuté par le UDF principal), pour ne pas considérer le code complet mais uniquement le code utilisé par ce UDF imbriqué.

**Corrections de bugs :**
- Correction de divers bugs avec l'éditeur de canevas et le mode de partage du canevas.
- Correction de divers bugs avec le menu de sélection de contexte du panneau AI et le bouton d'envoi.
- Correction du cache du moteur local lors du chargement des UDF avec `fused.load`.
- Correction du défilement des détails des travaux vers le haut lorsque de nouveaux journaux étaient disponibles.
- Correction de l'affichage de nombreux messages toast lors de l'enregistrement d'un canevas.
- Les jetons partagés et les travaux par lots chargeront désormais les UDF de la même manière que s'ils étaient exécutés par leur auteur d'origine.
- Correction de certains raccourcis Omnisearch cassés.
- Les UDF et arguments de `fused.submit` ne doivent plus être positionnels.
- Prévention de la récursion infinie lors de la détermination de la clé de cache pour `fused.cache`.
- Correction de l'utilisation d'un chemin Git incorrect pour récupérer le contenu des dépôts communautaires.
- La réflexion AI prend maintenant en charge le défilement automatique.
- Le panneau AI conserve l'invite lors du changement de modèle.
- Correction d'un bug où certaines instructions `print` pouvaient entraîner des journaux qui échouaient lors de l'envoi au Workbench.
- Ajustement du nombre d'UDF renvoyés par le serveur au Workbench à la fois.
