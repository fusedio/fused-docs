<summary>Pourquoi Python, alors qu'il existe SQL spatial ?</summary>

Python est le langage de référence pour la science des données spatiales. Bien que les jointures et transformations SQL spatiales puissent être effectuées efficacement en utilisant PostGIS dans une base de données externe, vous pourriez finalement avoir besoin de convertir ces données en Pandas et NumPy pour un traitement et une analyse supplémentaires, en particulier pour des opérations détaillées sur des tableaux raster. De plus, vous pouvez exécuter SQL directement sur Fused en utilisant des bibliothèques Python comme DuckDB, combinant ainsi les forces des deux approches.
</details>

<details>
<summary>Quel est l'avantage de la partition géographique des tables vectorielles ?</summary>

Cela permet une lecture efficace de grands ensembles de données en partitionnant stratégiquement les fichiers GeoParquet. Le format GeoParquet de Fused inclut des métadonnées qui permettent un filtrage spatial de tout ensemble de données, ne chargeant que les morceaux pertinents pour une zone d'intérêt spécifique. Cette approche réduit l'utilisation de la mémoire et vous permet de travailler avec des ensembles de données de n'importe quelle taille uniquement avec Python.
</details>

<details>
<summary>Comment puis-je créer un ensemble de tuiles qui couvrent une géométrie donnée ?</summary>

Des cas d'utilisation comme la création de chips peuvent nécessiter l'exécution d'une UDF sur un ensemble de tuiles qui tombent dans une géométrie donnée. Cela peut être fait en créant une liste de tuiles avec la bibliothèque [mercantile](https://github.com/mapbox/mercantile) puis en appelant la UDF [en parallèle](/core-concepts/run-udfs/run-small-udfs/#running-jobs-in-parallel-fusedsubmit).

```python showLineNumbers
import fused
import mercantile

bounds = [32.4203, -14.0933, 34.6186, -12.42826]

tile_list = list(mercantile.tiles(*bounds,zooms=[15]))
```
