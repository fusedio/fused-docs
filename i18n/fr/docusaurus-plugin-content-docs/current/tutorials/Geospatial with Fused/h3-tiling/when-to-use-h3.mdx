# Quand utiliser H3

Envisagez d'utiliser le carrelage H3 lorsque :
- Vous créez des cartes de chaleur d'événements (populations, comptes, etc.)
- Vous souhaitez comparer des ensembles de données à travers des résolutions et des échelles
- Vous travaillez avec des ensembles de données rares

```python showLineNumbers
common = fused.load("https://github.com/fusedio/udfs/blob/main/public/common/")
@fused.udf
def udf(bounds: fused.types.Bounds = [-122.71963771127753,36.53196328805067,-120.70395948802646,38.082911654639275]):

    releases = ['2024-02-15-alpha-0', '2024-03-12-alpha-0', '2024-08-20-0', '2024-09-18-0', '2024-10-23-0', '2024-11-13-0', '2024-12-18-0', '2025-01-22-0', '2025-03-19-1', '2025-04-23-0', '2025-05-21-0']
    release1 = releases[-1]
    df1 = common.read_hexfile(bounds, f"s3://fused-users/fused/sina/overture_overview/{release1}/hex{res}.parquet", clip=True)

    return df1

@fused.cache
def bounds_to_res(bounds, res_offset=0, max_res=14, min_res=3):
    z = common.estimate_zoom(bounds)
    return max(min(int(3 + max(0, z - 3) / 1.7 + res_offset), max_res), min_res)
```

### Conversion des ensembles de données en tuiles H3

Nous avons fourni quelques options pour convertir des ensembles de données en tuiles H3 :

| Type | Taille des données | Type de hexagonification | Sortie H3 | Considération |
|------|-------------------|-------------------------|-----------|---------------|
| [Fichier en H3](./file-to-h3) | Petit (< 100 Mo) | À la volée | Résolution statique | Facile à ajuster. Peut facilement changer le champ à hexagonifier |
| [Tuile dynamique en H3](./dynamic-tile-to-h3) | Petit (< 100 Mo) ou Grand (> 100 Mo) | À la volée | Résolution dynamique | Facile à ajuster. Peut facilement changer le champ à hexagonifier |
| [Ingestion d'ensemble de données en H3](./ingesting-dataset-to-h3) | Grand (> 100 Mo) | Pré-calculé | Résolution dynamique | Nécessite de décider à l'avance quelle variable hexagonifier |
