# Étape 1 : Chargement et Exploration des Ensembles de Données

## Zone d'Étude & Résolution H3

Nous utiliserons l'ensemble des États-Unis comme notre zone d'étude.

Cette analyse sera effectuée en utilisant des hexagones H3 pour joindre ces ensembles de données. Chacun de ces ensembles de données provient à l'origine de rasters. Nous les avons ingérés dans des hexagones H3, bien que nous devions décider d'une résolution H3 pour effectuer notre analyse.

Voici une comparaison des différentes résolutions H3 sur la même zone d'étude :

| Zone d'Intérêt      | Résolution Hex H3 | Taille Moy. de Cellule H3 | # Cellules H3 dans la Zone      |
|---------------------- |-------------------|----------------------|-------------------------|
| États-Unis Continentaux* | 4                 | ~1,770 km²             | ~10,000                 |
| États-Unis Continentaux* | 5                 | ~252 km²             | ~70,000                 |
| États-Unis Continentaux* | 6                 | ~36 km²              | ~49,000                 |

* Les États-Unis étant définis par les limites : `[-130, 25, -60, 50]`, c'est-à-dire environ 17,6M km² (les États-Unis font environ 9,8M km²)

Voir les [Statistiques de Comptage des Cellules H3 pour plus de détails](https://h3geo.org/docs/core-library/restable#cell-counts)

Nous avons décidé d'utiliser une **résolution H3 de 5** pour cette analyse.

:::info Règle générale H3
Augmenter une résolution d'un niveau réduit la surface de chaque hexagone par un facteur de 7. Cela signifie qu'à la résolution 5, chaque hexagone est 7x7 = 49x plus petit qu'à la résolution 3. 

Gardez cela à l'esprit car plus d'hexagones nécessitent également un temps de calcul plus long.
:::

<details>
<summary>Obtenir le nombre de cellules H3 pour les limites et la résolution</summary>

Nous pouvons écrire une simple UDF pour estimer le nombre de cellules H3 pour des limites et une résolution données.

```python
@fused.udf
def udf(bounds: fused.types.Bounds = [-130, 25, -60, 50], res: int = 5):
    
    # Charger la bibliothèque commune pour les utilitaires H3
    common = fused.load("https://github.com/fusedio/udfs/tree/56ec615/public/common/")
    
    # Convertir les limites en GeoDataFrame et calculer la surface UTM
    gdf_bounds = common.bounds_to_gdf(bounds)
    gdf_bounds = common.add_utm_area(gdf_bounds)
    area_millions = round(gdf_bounds['utm_area_sqm'].iloc[0] / 1e12, 1)
    print(f"Surface des limites : {area_millions}M kilomètres carrés")
    
    # Obtenir les hexagones H3 dans les limites à la résolution spécifiée
    df_hex = common.bounds_to_hex(bounds=bounds, res=res, hex_col="hex")
    print(f"{df_hex.T}")
    
    return df_hex
```

Renvoie :
```bash
>>> Surface des limites : 17.6M kilomètres carrés
>>>               0      ...               67634
hex  599309241731252223  ...  600345878259040255

[1 lignes x 67635 colonnes]
```

Ce résultat est de 17,6M km², c'est-à-dire environ 1,8 fois plus que la surface réelle des États-Unis (9,8M km²). 

C'est parce que les limites forment un rectangle _autour_ des États-Unis, et non la frontière exacte des États-Unis :

### Lecture des données dans UDF

- Charger l'ensemble de données hex en utilisant DuckDB depuis un `.parquet` sur S3

```python
# cdl_data_all_crops
@fused.udf
def udf():
    import duckdb

    df = duckdb.query(f"""
        SELECT * 
        FROM 's3://fused-asset/misc/hex/CDL_h12k1p1/year=2024/overview/hex5.parquet'
        and area>1000
    """).to_df()

    df = df.rename(columns={'data': 'crop_type'})

    return df
```

## Ensemble de Données 2 : Élévation

**Données**
- Tuiles Raster originales de [Terrain Tiles](https://registry.opendata.aws/terrain-tiles/)
- Ensemble de données partitionné en hex par Fused : `s3://fused-asset/data/dem/r6_z4_100k.parquet` (Voir sur [File Explorer](https://www.fused.io/workbench/files?path=s3%3A%2F%2Ffused-asset%2Fdata%2Fdem%2Fr6_z4_100k.parquet))

### Lecture des Données

Le fichier `.parquet` que nous allons utiliser n'est pas à la résolution hex 5, donc nous allons dynamiquement :
- Utiliser `h3_cell_to_parent(hex, 5) as hex,` pour convertir l'hex en l'hex parent à la résolution 5
- Utiliser `avg(mean_value) as mean_value` pour agréger par élévation moyenne de toutes les valeurs d'élévation couvrant l'hex parent
- Conserver uniquement les valeurs dans le viewport des limites avec `h3_cell_to_lng(hex)`

```python
@fused.udf
def udf(
    bounds:fused.types.Bounds=[-130, 25, -60, 50]
):
    # Chargement de la bibliothèque hex commune de Fused pour accéder à la bibliothèque H3 pré-installée dans DuckDB
    common = fused.load("https://github.com/fusedio/udfs/tree/56ec615/public/common/")
    con = common.duckdb_connect() 

    query = """
        SELECT  
            h3_cell_to_parent(hex, 5) as hex, 
            avg(mean_value) as mean_value 
        FROM read_parquet('s3://fused-asset/data/dem/r6_z4_100k.parquet')
        where 1=1
        and h3_cell_to_lng(hex) between ? and ? 
        and h3_cell_to_lat(hex) between ? and ? 
        group by 1
    """
    data = con.execute(query, [bounds[0], bounds[2], bounds[1], bounds[3]]).df()

    return data
```

## Ensemble de Données 3 : Température 

**Données**
- Tuiles Raster originales de [ERA5 Climate Reanalysis](https://registry.opendata.aws/era5-reanalysis-single-levels/)
- Ensemble de données partitionné en hex par Fused : `s3://fused-asset/data/era5/t2m_daily_mean_v2_1000/` (Voir sur [File Explorer](https://www.fused.io/workbench/files?path=s3%3A%2F%2Ffused-asset%2Fdata%2Fera5%2Ft2m_daily_mean_v2_1000%2Fmonth%3D))

### Lecture des Données

Nous allons créer une UDF qui prend :
- Une année et un mois donnés sous la forme `YYYY-MM`
- Une liste de limites 

Et renvoie :
- La température moyenne pour tout hexagone donné au cours de ce mois

Nous pouvons charger les données en utilisant DuckDB et les visualiser en utilisant la fonction `deckgl_hex` :

```python
@fused.udf
def udf(month='2024-05',
    bounds:fused.types.Bounds=[-130, 25, -60, 50]
):
    path = f's3://fused-asset/data/era5/t2m_daily_mean_v2_1000/month={month}/0.parquet'
    
    common = fused.load("https://github.com/fusedio/udfs/tree/56ec615/public/common/")
    con = common.duckdb_connect() 

    query = """
        SELECT  
            hex, 
            avg(daily_mean) - 273.15 as daily_mean 
        FROM read_parquet(?)
        WHERE 1=1
        AND h3_cell_to_lng(hex) between ? and ? 
        AND h3_cell_to_lat(hex) between ? and ? 
        GROUP BY 1
    """

    data = con.execute(
        query,
        [path, bounds[0], bounds[2], bounds[1], bounds[3]]
    ).df()

    print(f"{data.T=}")

    return data
```

Notez quelques points ici :
1. L'ensemble de données n'est pas à la résolution hex 5, c'est une résolution différente de celle des autres ensembles de données avec lesquels nous avons travaillé.
2. Les données ont beaucoup de zones "vides". 

C'est parce que les données sous-jacentes, l'ensemble de données ERA5, ont une résolution spatiale beaucoup plus grossière que les 2 autres ensembles de données (environ ~31 km à l'équateur, comparé à 30 m pour le Crop Data Layer et 5 m pour les données d'Élévation) 

Nous pouvons lisser les données à la résolution hex 5 en utilisant une agrégation de voisinage en k-ring :

```python
@fused.udf
def udf(month='2024-05',
    bounds:fused.types.Bounds=[-130, 25, -60, 50]
):
    common = fused.load("https://github.com/fusedio/udfs/tree/56ec615/public/common/")
    import pandas as pd  
    path = f's3://fused-asset/data/era5/t2m_daily_mean_v2_1000/month={month}/0.parquet'
 
    con = common.duckdb_connect() 

    query = """
        WITH base_data AS (
            SELECT  
                h3_cell_to_parent(hex, 5) as hex, 
                avg(daily_mean) - 273.15 as daily_mean 
            FROM read_parquet(?)
            WHERE 1=1
            AND h3_cell_to_lng(hex) between ? and ? 
            AND h3_cell_to_lat(hex) between ? and ? 
            GROUP BY 1
        ),
        all_hexes AS (
            -- Obtenir tous les hexes y compris les voisins en élargissant le k-ring
            SELECT DISTINCT unnest(h3_grid_disk(hex, 1)) as hex
            FROM base_data
        ),
        smoothed AS (
            -- Pour chaque hex (original + voisins), obtenir ses voisins en k-ring
            SELECT 
                ah.hex as hex,
                unnest(h3_grid_disk(ah.hex, 1)) as neighbor_hex
            FROM all_hexes ah
        )
        SELECT 
            s.hex,
            round(avg(b.daily_mean), 2) as daily_mean
        FROM smoothed s
        JOIN base_data b ON b.hex = s.neighbor_hex
        GROUP BY s.hex
    """ 

    data = con.execute(
        query,
        [path, bounds[0], bounds[2], bounds[1], bounds[3]]
    ).df()

    print(f"{data.T=}")

    return data
```
