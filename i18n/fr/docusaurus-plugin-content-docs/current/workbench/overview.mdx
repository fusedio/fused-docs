Workbench est une plateforme basée sur le navigateur pour travailler avec des données. Elle offre aux développeurs un accès immédiat à leurs données et un retour instantané lorsqu'ils ouvrent des fichiers, écrivent du code pour transformer des données et construisent des applications.

Les utilisateurs parcourent les ensembles de données dans l'[Explorateur de fichiers](/workbench/file-explorer/) et les ouvrent en tant que UDF, les convertissent en scripts d'analyse dans le [Constructeur de UDF](/workbench/udf-builder/), les partagent dans le [Catalogue de UDF](/workbench/udf-catalog), et enfin les transforment en applications partageables avec le [Constructeur d'applications](/workbench/app-builder/). Tous ces éléments tournent autour et interagissent avec les [Fonctions Définies par l'Utilisateur (UDF)](/core-concepts/why/) - des fonctions Python utilisées pour lire des fichiers, traiter des données et alimenter des applications.

Workbench prend en charge la création de flux de travail analytiques de bout en bout en quelques minutes, mais chacun de ses composants est entièrement remplaçable. L'Explorateur de fichiers peut être remplacé par des connexions de base de données en amont (comme [BigQuery](/tutorials/Geospatial%20with%20Fused/gee_bigquery/), [GEE](/tutorials/Geospatial%20with%20Fused/gee_bigquery/), et d'autres), et le Constructeur d'applications peut être remplacé par des applications tierces en aval (comme [DeckGL](/tutorials/Geospatial%20with%20Fused/other-integrations/#deckgl), Retool, et [d'autres](/tutorials/Geospatial%20with%20Fused/other-integrations/)).

Mais Workbench est plus qu'une simple plateforme pour travailler avec des données — c'est un portail vers une communauté où les créateurs partagent leurs UDF, applications et expertise pour s'aider mutuellement à améliorer leur travail.

Continuez à lire la documentation ou commencez avec Workbench sur [fused.io/workbench](https://www.fused.io/workbench).
