Le panneau de navigation sur le côté gauche du UDF Builder affiche les UDF disponibles pour l'utilisateur. Les UDF sont listés comme des couches. Leur ordre correspond à leur ordre d'empilement sur la carte. Chaque couche comprend des icônes pour supprimer, réorganiser et basculer la visibilité des UDF. La couche sélectionnée devient la couche "active". Son code apparaît dans l'[éditeur de code](/workbench/udf-builder/code-editor/) et sa sortie dans le panneau [Runtime](/workbench/udf-builder/runtime/).

## Zoom sur la couche

"Zoom sur la couche" vous permet de zoomer rapidement la carte sur l'emplacement de la couche UDF pertinente. Vous l'utiliseriez lorsque vous souhaitez concentrer la carte sur une zone spécifique ou des fonctionnalités définies par la couche, en particulier lorsque l'UDF a un composant spatial ou un état de vue par défaut. 
Les UDF qui retournent un objet avec un composant spatial ou ont un état de vue par défaut affichent un "zoom sur la couche". En cliquant dessus, la carte zoome sur l'emplacement pertinent.

## Visibilité de la couche

La fonctionnalité de basculement de couche permet aux utilisateurs d'afficher ou de masquer des visualisations spécifiques sur la carte. Ces couches fournissent des informations cartographiques améliorées, rendant les données spatiales plus faciles à comprendre et à interpréter.

## Options UDF

La liste des UDF comprend plusieurs options interactives pour gérer et organiser les fonctions de manière efficace.

### Glisser-déposer

Les utilisateurs peuvent réorganiser les UDF en les faisant glisser vers le haut et vers le bas dans la liste. Pour ce faire, ils doivent survoler le nom de l'UDF pour voir la poignée de glissement.

### Fermer/Supprimer

Si l'UDF est enregistré, un bouton "Fermer UDF" est disponible pour quitter l'éditeur. Si l'UDF n'est pas enregistré et est encore en cours d'édition, un bouton "Supprimer UDF" le remplace, permettant aux utilisateurs d'abandonner les modifications.

## Partager

Le bouton Paramètres dans le menu à trois points donne accès à toutes les options pour partager l'UDF.

![Share UDF](/img/workbench/udf-builder/new_share.png)

### Partager des extraits

Les UDF enregistrés dans le UDF Builder peuvent être appelés avec des [points de terminaison HTTPS](/core-concepts/run-udfs/run-small-udfs/#https-requests) en utilisant [le nom public de l'UDF](/core-concepts/run-udfs/run-small-udfs/#public-udf-name) et/ou [des tokens](/core-concepts/run-udfs/run-small-udfs/#token).

La section "Token partagé" montre des extraits pour exécuter l'UDF en utilisant un token public. Cela permet à toute application d'invoquer l'UDF sans authentification - y compris les appels `cURL`, Lonboard, Leaflet, [Mapbox](/tutorials/Geospatial%20with%20Fused/other-integrations/#mapbox), Google Sheets, DuckDB, le [App Builder](/workbench/app-builder/) de Fused, et les applications Python avec [`fused.run`](/core-concepts/run-udfs/).

### Extraits privés

Les extraits privés se trouvent désormais sous la section "Token partagé" dans la vue "Partager". La section "Extraits privés" montre des extraits qui ne peuvent être appelés que par des services authentifiés avec un token privé. Ceux-ci incluent `Links` et `Python snippets`.

#### Tags

Les tags UDF peuvent être définis pour aider à la découvrabilité dans le [Catalogue UDF](/workbench/udf-catalog/).

#### Description

Les UDF peuvent être documentés en utilisant [Markdown](https://www.markdownguide.org/basic-syntax/) avec une brève description de leur objectif, code et ensembles de données associés. La description apparaît dans le profil UDF et le fichier `README.md`.

## Barre d'outils

La barre d'outils sous le nom de l'UDF comprend des boutons pour Partager, Enregistrer et Télécharger. Des actions supplémentaires comme Dupliquer, Supprimer, Tirer des modifications et Pousser des modifications sont disponibles via le menu à trois points sur la droite.

### GitHub

Les organisations avec l'[Intégration GitHub](/core-concepts/content-management/git/#configuring-github-integration) activée peuvent pousser des UDF vers un dépôt GitHub en tant que [Pull Request](https://docs.github.com/en/pull-requests) ou restaurer une version antérieure d'un UDF à partir de l'historique des commits.

### Télécharger

Cliquer sur "Télécharger" télécharge un fichier `.zip` contenant le code UDF, le module et la configuration.
