Le catalogue UDF offre une collection consultable de [User Defined Functions (UDFs)](/core-concepts/why/) qui peuvent être importées dans le [UDF Builder](/workbench/udf-builder/) pour modification. Il facilite le partage et la découverte des UDF au sein des équipes et de la communauté Fused au sens large.

Les équipes de données rencontrent fréquemment des silos lorsqu'elles cherchent à partager et à réutiliser des extraits de code et des données à travers différents projets. Les notebooks traditionnels posent des défis en matière de contrôle de version, de partage de fonctions utilitaires individuelles, de gestion des dépendances et de reproduction des résultats dans de nouveaux environnements. De plus, les actifs finissent souvent déconnectés du code qui les a générés. Les UDF, en revanche, encapsulent des modules, des sorties et du code, ce qui aborde les problèmes liés à la reproductibilité et à la traçabilité.


- Créer un [nouveau UDF](/workbench/udf-catalog/#add-a-udf)
- [Rechercher](/workbench/udf-catalog/#search) dans le catalogue à travers les [catégories UDF](/workbench/udf-catalog/#udf-categories)
- Voir le [profil](/workbench/udf-catalog/#udf-profile) détaillé de chaque UDF
- Ajouter un UDF depuis son [URL GitHub](/workbench/udf-catalog/#add-from-github-url)
- Supprimer un UDF que vous avez créé vous-même
- Télécharger des UDF à partir de fichiers JSON ou ZIP

## Ajouter un UDF

Créez un nouveau UDF en cliquant sur le bouton "Nouveau UDF" en haut à droite. Cela ouvre un UDF avec un modèle de base. 
Alternativement, vous pouvez utiliser le bouton "Télécharger UDF" pour importer un UDF existant depuis votre machine locale.

![UDF Catalog](/img/udfcatalog_august.png)


## Recherche

Ouvrez le catalogue UDF en cliquant sur "UDF Catalog". Recherchez des UDF par nom, triez-les et basculez entre les vues galerie et liste. Cliquez sur les cartes UDF pour voir leurs profils ou les ajouter au UDF Builder.

![Add UDF Button](/img/addudfbutton.png)


### Catégories UDF

- **Tous les UDFs :** Voir tous les UDFs disponibles en un seul endroit, y compris toutes les catégories énumérées ci-dessous.
- **UDFs publics :** Vérifiés et accessibles à tous les utilisateurs
- **UDFs communautaires :** Partagés par la communauté et accessibles à tous les utilisateurs
- **UDFs d'équipe :** Partagés de manière privée au sein d'une équipe dans un dépôt [GitHub](/core-concepts/content-management/git/)
- **UDFs sauvegardés :** UDFs privés dans le compte de l'utilisateur


## Ajouter depuis l'URL GitHub

Si [l'intégration GitHub est activée](/core-concepts/content-management/git/#configuring-github-integration) sur votre Workbench, vous pouvez coller le lien de n'importe quel UDF depuis GitHub pour l'ouvrir. Cela vous permet d'ouvrir un UDF depuis n'importe quelle branche ou révision de GitHub.

## Profil UDF

Sélectionnez un UDF pour voir son profil qui inclut la [`README.md` description](/workbench/udf-builder/navigation/#description), la date de dernière mise à jour, l'auteur, le lien vers son code sur GitHub, et un aperçu du code.

### Par où commencer ?

Les UDFs peuvent être facilement réutilisés, donc avant de construire quelque chose de nouveau ou de partager le vôtre avec la communauté, consultez le [UDF Catalog](https://www.fused.io/workbench/catalog) pour voir si quelqu'un a déjà construit quelque chose dont vous pourriez bénéficier ! Voici quelques exemples d'UDFs publics et communautaires :

- [Ouverture des ensembles de données de bâtiments Overture](https://www.fused.io/workbench/catalog/Overture_Maps_Example-77970462-8c4d-44dd-87ad-3520f5d7bd83) pour parcourir le dernier ensemble de données de bâtiments des données ouvertes d'Overture sur [Source Coop](https://source.coop/repositories/fused/overture/description)
- [Exploration des images satellites Sentinel 2](https://www.fused.io/workbench/catalog/Sentinel_Tile_Example-a6b7839c-4886-4ea7-b168-c5c763c02c18) en utilisant [Microsoft's Planetary Computer](https://planetarycomputer.microsoft.com/dataset/sentinel-2-l2a) ou [AWS Open Data Program](https://registry.opendata.aws/sentinel-2/)
- [Calcul de la hauteur des bâtiments aux États-Unis en utilisant le modèle de surface numérique ALOS 30m](https://www.fused.io/workbench/catalog/DSM_Zonal_Stats-0c801e56-d0c4-47e8-a5ce-90d37703bdb7)

### Publier un UDF dans un dépôt GitHub

Une fois que vous avez écrit un UDF, vous pouvez utiliser le bouton Pousser vers GitHub dans Fused Workbench pour publier ou mettre à jour votre UDF dans un dépôt GitHub. Vous pourrez sélectionner le dépôt (par exemple, `fusedudf`, `community`, ou `public`) et créer automatiquement une demande de tirage sur GitHub. Lisez comment [activer l'intégration GitHub](/core-concepts/content-management/git/#configuring-github-integration).

Pour ajouter un UDF au dépôt communautaire, sélectionnez communauté dans le menu déroulant avant de créer une demande de tirage.
